Documentation

Everything you need to install, use, and contribute to taskbean.

Installation

taskbean runs locally on Windows 10/11. Choose the method that works for you.

Option 1: One-click launch

⌨terminal—□✕ 
git clone https://github.com/taskbean/taskbean.git
cd taskbean
./launch.cmd

launch.cmd handles everything: installs Foundry Local if needed, installs Python dependencies, registers the taskbean:// protocol handler, starts the server, and opens your browser.

Option 2: Manual setup

⌨terminal—□✕ 
# Install Foundry Local
winget install Microsoft.FoundryLocal

# Install Python dependencies
cd agent
pip install -r requirements.txt

# Launch
python main.py

Open http://localhost:2326 in your browser.

Quick Start

After launching, start typing in the chat input. There's no submit button — press Enter to send.

First launch: taskbean downloads and loads a default model (Phi-4 Mini). This takes a few minutes the first time. The status bar shows progress.

Try these commands:

  • "Add a task to review PRs by Friday"
  • "Remind me to stretch in 30 minutes"
  • "Plan my day" — AI prioritizes your existing tasks
  • "Drink water every 45 minutes" — creates a recurring reminder
  • Paste meeting notes — AI extracts action items automatically

Prerequisites

Required:

  • Windows 10 or 11
  • Python 3.10+ — python.org
  • Foundry Local — winget install Microsoft.FoundryLocal

Optional:

  • Docker — for Jaeger tracing (nerd mode)
  • Node.js 18+ — only needed for the legacy Node backend or Playwright E2E tests

AI Commands

taskbean uses natural language — there are no slash commands or special syntax. The AI understands intent and calls the right tool.

  • Add tasks — "add a task to fix the login bug"
  • Complete tasks — "mark the grocery task as done"
  • Set reminders — "remind me about the meeting at 2pm"
  • Recurring tasks — "drink water every 45 minutes"
  • Plan — "plan my day" or "what should I work on next?"
  • Extract — paste text or drop files, ask "what are the action items?"

Reminders

Reminders trigger Windows toast notifications. You can snooze or dismiss them from the notification itself.

  • One-time — "remind me to call the dentist at 3pm"
  • Relative — "remind me in 30 minutes"
  • Recurring — "stretch every 45 minutes during work hours"

Note: Notifications require win10toast (installed automatically). Configure Do Not Disturb hours in Settings.

File Extraction

Drop files or paste long text directly into the chat. taskbean uses MCP + MarkItDown to convert documents, then the AI extracts tasks.

Supported: PDFs, Word docs, emails, meeting notes, plain text, and more.

Model Switching

Click the model name in the header to open the model picker. You can switch between installed models without restarting.

  • The picker shows model size, compatibility, and performance
  • Models not yet downloaded show a download indicator
  • Switching is guarded — only one switch can happen at a time
  • The agent automatically rebuilds after every model switch

Themes

Four themes available in Settings: Dark Roast (dark), Latte (light), Espresso (warm), and Black Coffee (high contrast). Themes use CSS custom properties — no page reload needed.

Settings

Access via the gear icon. Settings persist to ~/.taskbean/config.json.

  • Model — select default model
  • Timezone — for time-aware reminders
  • Theme — visual appearance
  • Run on startup — register as Windows startup app
  • Notifications — enable/disable toast notifications
  • Model permission — "ask" (confirm downloads) or "yolo" (auto-download best model)

PWA Install

taskbean is a Progressive Web App. Install it from Edge or Chrome:

  1. Open http://localhost:2326
  2. Click the install icon (⊕) in the address bar
  3. The app installs as a standalone window with offline support

Nerd Mode

Click the Nerd button to open the telemetry panel. Four tabs:

  • Events — OTel span events with filter chips and search
  • Metrics — CPU, RAM, GPU, NPU sparkline cards
  • Traces — Waterfall trace view (proxied from Jaeger)
  • Logs — Severity-filtered log records

Running Tests

Python integration tests

⌨terminal—□✕ 
cd agent
pytest test_integration.py -v                # all tests
pytest test_integration.py -v -k test_health  # single test
pytest test_telemetry.py -v                  # telemetry tests

Playwright E2E tests

⌨terminal—□✕ 
npx playwright test                           # all 17 specs
npx playwright test tests/smoke.spec.js        # single spec
npx playwright test -g "page loads"             # by title
npx playwright test --project=smoke             # smoke only

Note: Playwright tests require the server running on :2326 and use Microsoft Edge.

Contributing

Contributions are welcome. The bean doesn't care who writes the code, only that it ships.

  • Python for the backend, vanilla JS for the frontend
  • No build step — edit and refresh
  • Playwright for E2E, pytest for integration tests
  • Keep tools in sync between agent/tools.py and server.js

See .github/copilot-instructions.md for detailed architecture notes and conventions.

Troubleshooting

"Server not running" overlay

The PWA needs the local Python server running. Start it with launch.cmd or cd agent && python main.py.

Model loading fails

⌨terminal—□✕ 
foundry model list                   # check installed
foundry model download phi-4-mini     # download manually

NPU not detected

NPU requires compatible hardware (Intel Core Ultra, Snapdragon X). taskbean falls back to GPU → CPU gracefully. Check hardware in Nerd Mode.

Port 2326 in use

⌨PowerShell—□✕ 
Get-NetTCPConnection -LocalPort 2326 | Select-Object OwningProcess
Stop-Process -Id <PID>

Voice input not working

Install a Windows speech language pack: Settings → Time & Language → Language & region → English (United States) → Speech recognition

Known Issues

  • NPU acceleration is Windows-only. GPU/CPU work cross-platform.
  • State is in-memory (Python) — resets on server restart.
  • First model download is 2–4 GB.
  • Single-user only — no multi-user support.
