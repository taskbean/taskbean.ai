How taskbean works

A local-first AI task manager built on on-device inference, real-time streaming, and zero-cloud architecture.

Key Concepts

The design decisions that make taskbean different.

On-Device Inference

The AI model runs directly on your hardware through Microsoft Foundry Local. It automatically detects your NPU, GPU, or CPU and selects the fastest execution provider. No API keys, no cloud calls, no latency penalty — just direct hardware access.

AG-UI Streaming

taskbean uses the AG-UI protocol for real-time agent communication. Every chat response streams as Server-Sent Events — you see tokens arrive, tools execute, and state update live. No polling, no WebSocket complexity.

Agent + Tools

The Python backend runs a real multi-turn agent with tool-calling. When you say "remind me to stretch in 30 minutes", the AI decides which tool to call, executes it, and streams the result. Not a wrapper around completions — a genuine agent loop.

Technology Stack

Intentionally simple. No webpack, no Docker requirement, no twelve microservices.

Frontend

Vanilla JS SPA

Single HTML file, CSS custom properties for theming, Lucide icons. No bundler, no framework, no build step. Edit and refresh.

Backend

Python · FastAPI

Agent Framework + AG-UI protocol. uvicorn ASGI server. All Foundry SDK calls run in threads to avoid blocking the event loop.

AI Runtime

Foundry Local

Microsoft's on-device inference SDK. Manages model downloads, execution provider registration, and hardware acceleration. NPU, GPU, CPU.

Telemetry

OpenTelemetry

Single OTel pipeline for traces, metrics, and logs. UISpanExporter fans out to the nerd panel. Optional Jaeger integration for deep tracing.

File Parsing

MCP + MarkItDown

Model Context Protocol handles document conversion. Drop PDFs, emails, meeting notes — MarkItDown converts to text, the AI extracts tasks.

Testing

Playwright + pytest

17 Playwright E2E specs in Edge. Python integration tests against a real uvicorn server on port 3001. No mocking — real inference, real assertions.

What happens when you send a message

From keystroke to completed task in milliseconds.

Input Routing

handleSend() checks the content. Short text goes to /api/command (AG-UI). Long text or file content goes to /api/extract for chunked processing.

Agent Execution

The Agent Framework runs a multi-turn loop. The AI decides: does it need a tool? It might call add_task, set_reminder, or just respond. Tools mutate in-memory state.

Live Streaming

Results stream back as AG-UI events: STATE_SNAPSHOT for the full todo list, STATE_DELTA for patches, and TEXT_MESSAGE_CONTENT for the AI's response.
